# additional-notes - szablon

Dodatkowe notatki do sync-audytu (logi z uruchomień, decyzje, checklisty). Dodawaj pliki `<ID-T>.md` (np. `01.md`, `02.md`).

Minimalny wzorzec jednego pliku:

```markdown
# <ID-T> - Krótki tytuł

## Surowe notatki / logi (lokalne)
...

## Wnioski dla kolejnych kroków
[HANDOFF: <lista ID-T, np. 02, 03>]
- ...
```
