# 01 – Doprecyzowanie widgetu dostępności

## Surowe notatki / logi (lokalne)

- Źródło danych: kalendarz projektowy (`calendar-service`), brak zapisu z GUI.
- Statusy do pokazania: `available`, `partial`, `off`; `partial` zawsze z godzinami (np. `09:00-13:00`).
- Priorytet widoku: tydzień, filtrowany po `teamId`; default to bieżący tydzień.

## Wnioski dla kolejnych kroków

[HANDOFF: 02, 03-01]

- Dla ID-T `02` (API):
    - Kontrakt `GET /availability?teamId=&week=` zwraca sloty godzinowe per dzień.
    - `partial` zawiera `hours` (ciąg znaków) i `comment` (np. „warsztat”).
    - Jeśli kalendarz nie ma danych dla dnia, API zwraca status `off`.
- Dla ID-T `03-01` (GUI):
    - Grid tygodnia musi renderować badge `available/partial/off`; `partial` pokazuje tooltip z `hours`.
    - Defaultowo użyj bieżącego tygodnia i ostatniego użytego `teamId`.
    - Wersja MVP nie pozwala edytować danych – tylko widok + tooltip.
