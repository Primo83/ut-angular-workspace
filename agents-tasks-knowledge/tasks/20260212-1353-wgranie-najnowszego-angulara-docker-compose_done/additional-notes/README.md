# additional-notes - szablon

Dodawaj pliki `<ID-T>.md` (np. `01.md`, `02-01.md`) gdy notatki nie mieszczą się w `tasks.md`. Trzymaj tu logi, SQL, decyzje architektoniczne i rozszerzone wnioski.

Minimalny wzorzec jednego pliku:

```markdown
# <ID-T> - Krótki tytuł

## Surowe notatki / logi (lokalne)
...

## Wnioski dla kolejnych kroków
[HANDOFF: <lista ID-T, np. 02, 03-01>]
- ...

## Powiązane zadania
- Kolejne: ...
- Ewentualne: ...

## Linki / referencje
- ...
```
